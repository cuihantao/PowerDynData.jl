PowerDynData.DEBUG_LEVEL — Constant
DEBUG_LEVEL

Global debug level reference. Set to 0 to disable all debug output. Levels:

  • 0: No debug output
  • 1: Basic parsing progress
  • 2: Detailed field parsing
  • 3: Verbose byte-level operations

PowerDynData.DynamicData — Type
struct DynamicData

Top-level container for all dynamic data from a DYR file.

Fields

  • models::Dict{String, DynamicRecords}: Dictionary mapping model names to their records

  • metadata_registry::Union{Nothing, MetadataRegistry}: Metadata registry (if loaded)

  • source_file::String: Source file path

  • validation_issues::Vector{ValidationIssue}: Validation issues encountered during parsing

Examples

dd = parse_dyr("case.dyr", metadata_dir="metadata")

# Access models
genrou = dd["GENROU"]
tgov1 = dd["TGOV1"]

# Get available models
keys(dd)

# Check validation issues
if !isempty(dd.validation_issues)
    for issue in dd.validation_issues
        println("$(issue.model_name)[$(issue.record_index)].$(issue.field_name): $(issue.message)")
    end
end

# Convert to DataFrame
using DataFrames
df = DataFrame(dd["GENROU"])

struct FieldMetadata

Metadata for a single field in a model.

Fields

  • name::Symbol: Field name

  • position::Int64: Position in DYR record (1-indexed)

  • type::Type: Julia type for this field

  • description::String: Human-readable description

  • unit::String: Physical unit (e.g., 'seconds', 'MW', 'dimensionless')

  • required::Bool: Whether this field is required

  • default::Any: Default value if not provided

  • range::Union{Nothing, Tuple{Float64, Float64}}: Valid range for numeric fields

PowerDynData.ModelMetadata — Type
struct ModelMetadata

Complete metadata for a PSS/E dynamic model.

Fields

  • name::String: Model name (e.g., 'GENROU')

  • description::String: Description of the model

  • category::String: Category (e.g., 'generator', 'exciter', 'governor')

  • model_name_field::Int64: Position of model name field in DYR record

  • multi_line::Bool: Whether record spans multiple lines

  • line_count::Union{Nothing, Int64}: Number of lines (if multi_line)

  • terminator::String: Record terminator character

  • flexible_fields::Bool: Allow variable number of fields

  • fields::Vector{PowerDynData.FieldMetadata}: Field metadata for all fields in order

PowerDynData.parse_dyr — Method
parse_dyr(source; metadata_dir=pkgdir(PowerDynData, "metadata")) -> DynamicData

Parse a PSS/E DYR file into structured data.

Arguments

  • source: Path to DYR file or IO object
  • metadata_dir: Path to metadata directory. Defaults to bundled metadata at pkgdir(PowerDynData, "metadata"). Set to nothing to disable metadata and use indexed fields only. Can be set to a custom path for user-defined metadata.

Returns

  • DynamicData: Container with all parsed models

Examples

# With bundled metadata (default - recommended)
dd = parse_dyr("case.dyr")

# Access models with named fields
using DataFrames
genrou_df = DataFrame(dd["GENROU"])

# Without metadata (indexed fallback)
dd = parse_dyr("case.dyr", metadata_dir=nothing)

# With custom metadata directory
dd = parse_dyr("case.dyr", metadata_dir="path/to/custom/metadata")

PowerDynData.@pdebug — Macro
@pdebug level msg

Print debug message if DEBUG_LEVEL[] >= level.

Named @pdebug to avoid conflicts with Julia's built-in @debug macro.

Examples

PowerDynData.DEBUG_LEVEL[] = 1
@pdebug 1 "Parsing model: GENROU"  # Prints
@pdebug 2 "Field value: $(val)"    # Does not print
